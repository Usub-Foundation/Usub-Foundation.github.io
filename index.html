Philosophy

We move with
the standard.

Most C++ libraries stagnate at C++14 or C++17, held back by compatibility concerns or a lack of conviction. We don't believe in writing tomorrow's infrastructure with yesterday's tools.

C++17 2017 behind us
The baseline most of the C++ world still targets. Solid, but missing the language features that make async truly ergonomic. We built past it.
structured bindings if constexpr std::optional std::variant
C++20 2020 foundation
Coroutines landed. This is what made the entire uvent architecture possible — co_await as a first-class language feature, not a library hack on top of callbacks.
coroutines · co_await concepts ranges std::span
C++23 2023 current standard
The entire stack is written to C++23. Not as a marketing claim — we use std::expected, deducing this, and improved ranges throughout. This is the minimum we ship against today.
std::expected deducing this std::print improved ranges std::flat_map
C++26 est. 2026 we'll be there
When compiler support is solid and the standard is ratified, we migrate. Not immediately — we wait for real toolchain support, not paper proposals. But we don't wait out of fear.
reflection (P2996) contracts sender/receiver (P2300) inplace_vector
What we reject
✕ Compatibility as an excuse. Supporting C++11 because someone might need it is how ecosystems calcify.
✕ Macro-based abstractions. ureflect does compile-time reflection with zero macros. That's not a coincidence.
✕ Dependencies as defaults. If the STL can do it, we don't add a dep. Every dep is a liability.
✕ Freezing at "good enough." The unet rewrite exists because the old design had a ceiling. We hit it and rewrote.
What we commit to
✓ The current standard, always. C++23 today. C++26 when toolchains are ready. Not before, never after.
✓ Rewrites over patches. When the architecture has a ceiling, we rebuild — not paper over it.
✓ Idiomatic over clever. Code that reads like modern C++ is easier to adopt, audit, and contribute to.
✓ Performance with principles. Zero-cost abstractions aren't a slogan — every design decision gets benchmarked.
A note on C++26 reflection

P2996 — the proposal to add static reflection to the standard — is essentially formalising what ureflect already does today via C++23 techniques. When it lands, the ecosystem is positioned to align with or inform the standard rather than play catch-up.

Investment thesis

Why this, why now.

C++ isn't going anywhere — it's accelerating. AI inference, financial systems, game engines, embedded. The tooling just hasn't caught up.

01
The entrypoint problem
Starting a new C++ networked application today means choosing between Boost (enormous, complex), libuv (C API), or rolling your own. Usub offers a third path: idiomatic C++23 coroutines with modern ergonomics and zero external overhead.
02
A composable, coherent stack
uvent → unet → upq/uredis/urpc is a complete backend stack. ureflect powers ujson and ulog. umath handles precision-critical arithmetic. These aren't isolated libraries — they're designed to compose, with consistent APIs throughout.
03
The C++23 timing window
C++23 coroutines are standardized but ecosystem support is nascent. Usub is building the ergonomic layer at exactly the right moment — before incumbents react, while compiler support is finally solid across MSVC, Clang, and GCC.
Roadmap

What's being built.

Active development across the stack, with unet's strict-parsing rewrite as the current flagship effort.

Done
Core async runtime (uvent v3.7)
Deterministic thread routing, async barriers, io_uring backend, IOCP on Windows, kqueue on BSD. 16 releases shipped.
Done
Reflection + data layer
ureflect, ujson, upq, uredis, ulog — the full data access and serialization layer, all reflection-aware and zero-boilerplate.
Active
unet strict-parsing rewrite
The strict_parsing branch is a significant leap over stable main — full RFC-compliant HTTP parsing, cleaner internals. The current active focus.
Planned
HTTP/2 + TLS (unet)
HTTP/2 multiplexing and native TLS integration to bring unet to production-grade coverage for modern web services.
Planned
Package distribution + docs
vcpkg / Conan packages, expanded documentation, and example application templates to lower the barrier for new adopters.
Planned
C++26 migration
When toolchain support is solid — particularly for P2996 static reflection — the stack migrates forward. ureflect is already positioned to align with the incoming standard.
Planned
Community + enterprise support
Contributor onboarding, professional support tiers, and commercial licensing options for enterprise integrations.
